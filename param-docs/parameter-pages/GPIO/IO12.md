# Parameter `IO12`
Default Value: `input-pullup disabled 10 false false`

!!! Warning
    This is an **Expert Parameter**! Only change it if you understand what it does!

This parameter can be used to configure the GPIO `IO12` pin.

!!! Danger "CRITICAL WARNING: Boot Strapping Pin Conflict"
    **GPIO12 is a strapping pin that determines flash voltage at boot!**
    
    **DO NOT use GPIO12 for I²C SDA, I²C SCL, or 1-Wire (OneWire) modes with pull-up resistors!**
    
    The pull-up resistor (required for I²C and 1-Wire sensors) will hold GPIO12 HIGH during boot, 
    causing the ESP32 to switch to 1.8V flash mode and **preventing boot on standard 3.3V flash modules**.
    
    **Symptom**: Device does not boot when sensor is connected, boots fine when disconnected.
    
    **Boot error logs** may show: `invalid header: 0xffffffff` or `ets_main.c`
    
    **Solution**: Use GPIO1 or GPIO3 instead (requires disabling USB logging).
    
    See [I²C SDA Configuration](I2C-SDA.md) and [1-Wire Configuration](OneWire.md) for safe alternatives.

!!! Note
    This pin can be used for digital I/O without pull-ups, but is NOT recommended for sensor interfaces 
    that require pull-up resistors (I²C, 1-Wire).

Parameters:

- `GPIO 12 state`: One of `external-flash-ws281x`, `input`, `input pullup`, `input pulldown` or `output`.
- `GPIO 12 use interrupt`: Enable interrupt trigger
- `GPIO 12 PWM duty resolution`: LEDC PWM duty resolution in bit
- `GPIO 12 enable MQTT`: Enable MQTT publishing/subscribing
- `GPIO 12 enable HTTP`: Enable HTTP write/read
- `GPIO 12 name`: MQTT topic name (empty = `GPIO12`). Allowed characters: `a-z, A-Z, 0-9, _, -`.
