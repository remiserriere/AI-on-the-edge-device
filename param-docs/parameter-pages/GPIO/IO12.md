# Parameter `IO12`
Default Value: `input-pullup disabled 10 false false`

!!! Warning
    This is an **Expert Parameter**! Only change it if you understand what it does!

This parameter can be used to configure the GPIO `IO12` pin.

!!! Info "GPIO12 Restrictions"
    **Boot Strapping Pin:** GPIO12 determines flash voltage at boot (LOW=3.3V, HIGH=1.8V).
    
    **Critical Timing Distinction:**
    - ❌ **Hardware pull-ups** (physical resistors) - Pull GPIO12 HIGH **during boot** → boot failure
    - ✅ **Software pull-ups** (input-pullup mode) - Enabled **after boot** → safe to use
    
    **Not Available for Sensor Modes:**
    - ❌ I²C (i2c-sda, i2c-scl) - requires hardware pull-up resistors → boot failure
    - ❌ 1-Wire (onewire) - requires hardware pull-up resistor → boot failure
    
    **Available Modes:**
    - ✅ WS2812 LEDs (external-flash-ws281x) - no pull-ups needed, idle LOW, safe
    - ✅ Basic I/O (input, output, input-pullup*) - software pull-up enabled after boot
    
    \* **input-pullup mode is safe** because ESP32's internal pull-up is configured by software after boot completes, so GPIO12 reads LOW during the boot strapping sample.
    
    **Why sensors fail:** I²C and 1-Wire require physical 4.7kΩ resistors soldered/connected between GPIO12 and 3.3V. These resistors pull GPIO12 HIGH before ESP32 even powers up, causing wrong flash voltage selection at boot.
    
    **Why WS2812 works:** Actively driven data (push-pull), no hardware pull-ups, idle state LOW.
    
    **For sensors:** Use GPIO1 or GPIO3 (requires disabling USB logging).

Parameters:

- `GPIO 12 state`: One of `external-flash-ws281x`, `input`, `input pullup`, `input pulldown` or `output`.
- `GPIO 12 use interrupt`: Enable interrupt trigger
- `GPIO 12 PWM duty resolution`: LEDC PWM duty resolution in bit
- `GPIO 12 enable MQTT`: Enable MQTT publishing/subscribing
- `GPIO 12 enable HTTP`: Enable HTTP write/read
- `GPIO 12 name`: MQTT topic name (empty = `GPIO12`). Allowed characters: `a-z, A-Z, 0-9, _, -`.
